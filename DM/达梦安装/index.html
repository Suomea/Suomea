    文件安装

    准备工作

    单独创建用户和组安装达梦。 组：dinstall 用户：dmdba

    安装目录：/home/dmdba/dmdbms 实例保存目录：/dmdata/data

    创建用户和组

    创建组和用户，不能使用 root 用户安装。 

    groupadd dinstall -g 2001
useradd -G dinstall -m -d /home/dmdba -s /bin/bash -u 2001 dmdba

    groupadd - g 参数指定了群组的 GID。

    useradd - -G 指定了账号的所属群组。 - -m 强制建立用户的家目录。 - -d 指定了用户的家目录。 - -s 指定用户登录后的 shell。 - -u 指定用户的 UID。

    设置 dmdba 用户的密码 

    passwd dmdba

    创建数据存储目录 

    mkdir -p /dmdata/data
chown -R dmdba:dinstall /dmdata/data
chmod -R 755 /dmdata/data

    资源限制设置

    设置 dmdba 用户的系统资源限制 编辑配置文件/etc/security/limits.conf，新增如下内容： 

    dmdba  soft      nice       0
dmdba  hard      nice       0
dmdba  soft      as         unlimited
dmdba  hard      as         unlimited
dmdba  soft      fsize      unlimited
dmdba  hard      fsize      unlimited
dmdba  soft      nproc      65536
dmdba  hard      nproc      65536
dmdba  soft      nofile     65536
dmdba  hard      nofile     65536
dmdba  soft      core       unlimited
dmdba  hard      core       unlimited
dmdba  soft      data       unlimited
dmdba  hard      data       unlimited

    设置资源限制的用户为 dmdba，这个限制会对属于该用户的所有进程都生效。

    hard 为严格的设定，必定不能超过这个设定的数值。 soft 为警告的设定，可以超过这个数值，但是若超过会有警告信息。一般 soft 设置小于 hard。

    nice 设置进程调度的优先级为 0（默认值）。 as 虚拟内存设置无限制。 fsize 文件大小无限制。 nproc 设置用户可以启动的最大进程数。 nofile 设置用户可以同时打开的文件数量。 core 核心转储文件大小无限制，即进程在崩溃路时生成的核心转储文件可以是任意大小的。 data 进程可以使用的数据段的大小没有限制。

    配置环境变量

    编辑 /home/dmdba/.bash_profile，内容如下： 

    export LD_LIBRARY_PATH="$LD_LIBRARY_PATH:/home/dmdba/dmdbms/bin"
export DM_HOME="/home/dmdba/dmdbms"
export PATH=$PATH:$DM_HOME/bin:$DM_HOME/tool

    切换至 dmdba 用户，使配置生效。 

    # su dmdba
$ source .bash_profile

    启动

    启动数据库 

    $ cd /home/dmdba/dmdbms/bin
$ ./DmServiceDM start
Starting DmServiceDM:                                      [ OK ]

    停止数据库 

    $ ./DmServiceDM stop

    重启数据库 

    $ ./DmServiceDM restart

    查看数据库状态 

    $ ./DmServiceDM status
